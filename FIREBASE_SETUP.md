# 🔥 Firebase Setup Guide for Yabalitsa Management

This guide will help you set up Firebase for your Yabalitsa Management system.

## ⚠️ Important: Collection Naming

**Your existing Firebase project already has these collections:**
- `games`
- `notification_logs` 
- `notification_requests`
- `users`
- `venues`

**The Yabalitsa Management system will create NEW collections with `yabalitsa_` prefix:**
- `yabalitsa_venues` - Football venue information
- `yabalitsa_pitches` - Individual pitch details
- `yabalitsa_bookings` - Booking records
- `yabalitsa_customers` - Customer information (renamed from users)
- `yabalitsa_timeSlots` - Available time slots
- `yabalitsa_blockedSlots` - Blocked/unavailable slots
- `yabalitsa_venueSettings` - Venue-specific configurations

**This ensures NO CONFLICTS with your existing app!**

## 📋 Prerequisites

- Firebase project already created (yabalitsa-6f5e8)
- Firebase CLI installed (optional, for deployment)

## 🚀 Step-by-Step Setup

### 1. Enable Firestore Database

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Select your project: `yabalitsa-6f5e8`
3. In the left sidebar, click **Firestore Database**
4. Click **Create Database**
5. Choose **Start in test mode** (for development)
6. Select a location close to your users (e.g., `europe-west1`)
7. Click **Done**

### 2. Enable Authentication (Optional for now)

1. In the left sidebar, click **Authentication**
2. Click **Get started**
3. Go to **Sign-in method** tab
4. Enable **Email/Password** provider
5. Click **Save**

### 3. Set Up Firestore Security Rules

1. In Firestore Database, go to **Rules** tab
2. Replace the default rules with the content from `firestore.rules` file
3. Click **Publish**

**The rules will:**
- ✅ Allow access to all `yabalitsa_*` collections for authenticated users
- ✅ Preserve access to your existing collections (`games`, `users`, `venues`, etc.)
- ✅ Keep your existing app working exactly as before

### 4. Collection Creation

Firestore will automatically create the new `yabalitsa_*` collections when you first add documents. Your existing collections remain untouched.

### 5. Test the Connection

1. Start your development server: `npm run dev`
2. Go to `/venues/new` and try to create a venue
3. Check Firestore Database to see if the document was created in `yabalitsa_venues`

## 🔧 Configuration Details

### Firestore Rules
The security rules allow authenticated users to read and write to all `yabalitsa_*` collections while preserving your existing collection access.

### Data Structure
Each document will have:
- `createdAt` - Server timestamp when created
- `updatedAt` - Server timestamp when last modified
- Document ID - Auto-generated by Firestore

## 🚨 Important Notes

1. **No Conflicts**: The `yabalitsa_` prefix ensures your existing app continues to work unchanged
2. **Test Mode**: Starting in test mode means anyone can read/write to your database. This is fine for development but not for production.
3. **Location**: Choose a Firestore location close to your users for better performance.
4. **Indexes**: Firestore will automatically create indexes for simple queries. Complex queries may require manual index creation.
5. **Quotas**: Be aware of Firestore quotas and pricing for production use.

## 🔒 Production Security

When moving to production:

1. **Custom Claims**: Implement user roles and permissions
2. **Structured Rules**: Create more restrictive security rules for `yabalitsa_*` collections
3. **Data Validation**: Add server-side validation
4. **Monitoring**: Set up Firebase monitoring and alerts

## 🆘 Troubleshooting

### Common Issues

1. **Permission Denied**: Check if Firestore rules are published
2. **Collection Not Found**: `yabalitsa_*` collections are created automatically when first document is added
3. **Network Errors**: Verify your Firebase config in `src/lib/firebase.ts`
4. **Existing Data**: Your existing collections (`games`, `users`, `venues`) remain completely untouched

### Getting Help

- Check Firebase Console for error logs
- Verify your Firebase configuration
- Ensure `yabalitsa_*` collections exist in Firestore
- Check browser console for JavaScript errors
